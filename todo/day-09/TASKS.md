# Day 09 — Create / edit job posting

## Goal
EMPLOYERs can create, edit, and save-as-draft a job posting, gated by company verification status (warning, not block).

## Depends on
Day 08.

## Tasks
- [ ] Build `/employer/jobs/new/page.tsx` form: title, location, remote dropdown, employment type, experience, salary min/max + currency, tags (multi-input), `expires_at` (default +30d), description (Tiptap), DRAFT/OPEN toggle.
- [ ] Server Action `createJob`: `requireRole(EMPLOYER)`, `requireVerified()`, sanitize HTML, generate unique slug from title, validate salary range, insert.
- [ ] Build `/employer/jobs/[id]/edit/page.tsx` reusing the same form.
- [ ] Server Action `updateJob` + `closeJob`.
- [ ] Build `/employer/jobs/page.tsx` listing this employer's jobs with status, applicant count placeholder, edit/close actions.
- [ ] Show a yellow banner on the form when the company is unverified: "Your job will post but won't show a Verified badge until you verify."

## Files created/changed
- `app/(employer)/employer/jobs/new/page.tsx`
- `app/(employer)/employer/jobs/[id]/edit/page.tsx`
- `app/(employer)/employer/jobs/page.tsx`
- `app/(employer)/employer/jobs/actions.ts`
- shadcn: `command` (for tags multi-select), `calendar` + `popover` (for date picker).

## Done when
- Creating a job inserts a row with the right slug, status, and HTML.
- Salary min > max fails validation server-side.
- Editing a job updates the row, including `searchVector` (auto-regenerated by Postgres).
- Closing a job flips status to `CLOSED` and hides it from the public list (verified next day).
